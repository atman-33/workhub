# vault-template OpenCode runtime

- After editing any `.ts` under `vault-template/.opencode/plugins/`, run
  `npm run check:opencode` and confirm it passes. After editing any `.mjs`
  under `.opencode/scripts/` or `.opencode/mcp/`, run `node --check <file>`.
- These files are deployed into user vaults (copy-if-missing via
  `init_vault`) and run there with **no node_modules**: plugins may use
  type-only imports from `@opencode-ai/plugin` (erased at runtime) and Node
  built-ins, never runtime npm dependencies.
- `.opencode/skills/` in a vault is generated by
  `scripts/sync-claude-skills.mjs` from the enabled Claude plugins — never
  ship skills in `vault-template/.opencode/`.
- The serena MCP launcher must keep `--python 3.11` (3.12+ breaks with an
  OpenSSL error).
- Keep the Claude Code hook scripts (`plugins/engineering/hooks/scripts/*.mjs`)
  and their OpenCode mirrors (`vault-template/.opencode/plugins/*.ts` +
  `lib/project-context-core.ts`) behaviorally aligned — a matching-logic
  change in one side must land in both.
